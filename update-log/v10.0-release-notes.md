# v10.0 Release Notes — 风格切片复刻系统 & 规划流程精简

**版本**: 10.0.0  
**发布日期**: 2026-04-08

---

## 核心变更

### 1. 规划流程精简（Phase A）

**原9步压缩为3步**：快速立项 → 一页大纲 → 直接开写

- **新增**：`快速立项（project_card）`步骤 — 一次访谈（最多2轮问答）采集核心卖点、主角基线、世界观框架、关键角色
- **删除**：步骤1（一句话概括）、步骤2（当前卷落点）、步骤2.5（世界观体系）、步骤3（一页人物介绍）
- **删除**：步骤5（人物大纲）、步骤6（四页大纲）
- **保留**：步骤0（对标分析，可选）、步骤4（一页大纲）、步骤7-11

### 2. 场景类型接口协议（Phase B）

- Layer-1 轻量场景卡新增两个字段：
  - `scene_type`（必填）：`combat` / `face_slap` / `negotiation` / `emotional` / `reveal` / `daily` / `system`
  - `scene_intensity`（可选，默认 medium）：`high` / `medium` / `low`
- 步骤8改为"章前规划"模式：每次只规划下一章，不再生成整卷场景清单
- ToT推演从强制阻断改为建议

### 3. 风格切片复刻系统（Phase C+D）

- 新建 `scripts/slice_library.py` — 从完整小说文本构建按场景类型分类的原文切片库
- `scripts/analyze_dna.py` 新增 `--chapter-map` 模式，输出章节类型标注
- 切片库存储在全局路径 `~/.narrativespace/style_library/{author_slug}/`，跨项目复用
- 写作模块（writing/interactive）新增风格切片加载逻辑：根据 `scene_type` + `scene_intensity` 自动选片注入

### 4. 基础设施更新（Phase E）

- `scripts/init.py` 新增 `--link-author` 参数，可在初始化时绑定切片库作者
- `templates/state_template.json` 新增 `benchmark_state.linked_author` 和 `style_library_path` 字段

### 验证结论

**通过**。四个回归场景均验证通过：

- **场景A**：linked_author 已绑定时，Step 10 正确注入语感切片（含 scene_type 匹配、daily 回退、YAML 头剥离）
- **场景B**：linked_author 为空时，Step 10 正常降级，不报错
- **场景C**：Step 7/8/9/11 全部使用正式模板，无骨架降级
- **场景D**：meta_08/09/10 规则提取有实际文本，无空壳；meta_02 空壳有明确 WARNING

### 残留风险

- `meta_02` 为容器节点（其子规则 ps_01~ps_05 使用 `rule` 字段），顶层 meta_02 本身无文本字段，将持续输出 WARNING。后续可考虑将 step_rule_map.json 中 meta_02 的引用替换为具体子规则 ID。
- `_load_style_snippet` 依赖 PyYAML；若环境无 PyYAML，函数静默返回空字符串（安全降级，但不会提醒安装）。

### 后续建议

1. 为 `meta_02` 的子规则（ps_01~ps_05）在 `step_rule_map.json` 中建立精确 ID 映射。
2. 在 `init.py` 启动检查中验证 PyYAML 是否可用，如缺失则提示安装。
3. Step 8 模板目前生成整卷场景清单，与 v10.0 release notes 中"改为按章生成"的描述存在差异，建议后续按章前规划重构 step_8.md。
