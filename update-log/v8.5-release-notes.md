## 新特性 / New Features

### Stream 1 — 全局存储 + 级联渲染 / Global Storage + Cascade Rendering

| 组件 | 说明 |
|------|------|
| `~/.narrativespace/global_author_dna.yaml` | 全局作者 DNA 文件（跨项目持久层） |
| `.xushikj/config/global_author_dna.yaml` | 项目本地副本（自动同步） |
| `.xushikj/style_logs/cycle_quirks.md` | 本卷短期覆写层（最高优先级） |
| `config/archived_dna.yaml` | 衰减规则归档库 |

**优先级链（高→低）:**
```
cycle_quirks（尾部） > dna_human_*.yaml > clone_*.yaml > style_modules > global_author_dna（基底） > 内置规则
```

**三层级联加载（写作/交互模式均支持）:**
1. 加载全局 DNA 作为基底约束
2. 项目级 dna_human_*.yaml 自动覆写同类规则
3. 本卷 cycle_quirks.md 最后注入，获得最高注意力权重

### Stream 2 — 风格路由器 / Style Router

新增 `/style` 指令（写作与交互模式均支持）：

```
用户: /style 本章对话太正式了，需要更口语化
```

LLM 生成结构化 JSON → 保存到 `style_actions/pending_001.json` → 运行 `update_style_rule.py` 落盘。

**作用域选项:**
- `global` — 永久写入全局 DNA（所有项目生效）
- `project` — 写入 `.xushikj/config/dna_human_*.yaml`（当前项目）
- `cycle` — 写入 `.xushikj/style_logs/cycle_quirks.md`（本卷结束即归档）

### Stream 3 — DNA 记忆蒸馏 / Memory Distillation

`consolidate_dna.py` 执行六阶段提纯：

| 阶段 | 操作 |
|------|------|
| A | 加载全局 DNA |
| B | 语义去重（Ollama 余弦相似度 > 0.85，降级到精确匹配） |
| C | 冲突检测（极性分析，标记人工确认项，不自动处理） |
| D | 权重衰减（长期未触发规则 -20%，低于阈值归档） |
| E | Token 预算强制执行（超出 50 条按 weight×recency 排序归档） |
| F | 写回全局文件 + 本地副本 + 更新 archived_dna.yaml |

```
# 每卷结束时触发提纯
python volume_snapshot.py --volume 3 --consolidate-dna
```

## 工程决策 / Engineering Decisions

### 为什么用文件而不是 CLI `--json` 传参？

Windows Shell 对 `--json '{"key": "val"}'` 的引号转义复杂且不稳定。
使用 `style_actions/pending_*.json` 文件中转，彻底消除转义问题。

### 为什么语义去重不自动合并？

合并规则需要语义判断（取哪条措辞？保留哪个 weight？），自动合并存在歧义。
系统仅**标记**候选重复，输出报告供用户确认，不自动删除。

**例外:** 极性冲突（禁止 vs 必须语义相反的相似规则）强制要求人工处理。

### 为什么 Token 预算是强制而非建议？

超出预算的规则在推理时会被 LLM 遗忘（注意力稀释），不如主动归档低权重规则，
确保每次推理都在高效的预算范围内。

## 升级说明 / Upgrade Instructions

```bash
# 在项目目录下运行（自动执行 patches 18-21，创建新目录）
python scripts/init.py --upgrade

# 初始化全局 DNA（首次使用）
mkdir -p ~/.narrativespace
cp templates/global_author_dna_template.yaml ~/.narrativespace/global_author_dna.yaml
```

## 破坏性变更 / Breaking Changes

无。v8.5 全部为向后兼容的新增功能。全局 DNA 文件不存在时，
级联加载会静默跳过，不影响现有项目使用。

No breaking changes. v8.5 is fully backward compatible. When the global DNA file
does not exist, cascade loading silently skips it without affecting existing projects.
