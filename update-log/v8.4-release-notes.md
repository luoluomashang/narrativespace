## <a id="1"></a>1. 人味密度扩充（ht_08 ~ ht_12）

### 中文说明

**配置文件：** `config/human_touch_rules.yaml`  
**影响模块：** `modules/writing/prompt.md`、`modules/interactive/prompt.md`

v8.2 仅有 ht_01 ~ ht_07 七条人味写作规则。v8.4 新增五条，将规则总数扩充至 12 条。

| 规则 ID | 名称 | 核心要求 |
|--------|------|---------|
| ht_08 | 记忆碎片法 | 角色触景生情时植入非线性闪回，约 200 字内，以感官锚点触发 |
| ht_09 | 渺小-宏大对比 | 每万字至少一处用日常微观事物映射宏大主题（如：吃包子→战场厮杀） |
| ht_10 | 哲学隐喻 | 角色行为或场景自然生发出世界观洞察，禁止直白总结 |
| ht_11 | 人性缺口暗示 | 通过回避行为/话题暗示角色的未愈创伤，不得直接揭示 |
| ht_12 | 意象回响 / Motif Echo | 开头植入的意象/词语在高潮处回响，形成闭环共鸣 |

**附加子规则：**
- `ht_04_ext_a`（对话错位 dialogue_mismatch）：角色对话与内心独白形成反差
- `ht_04_ext_b`（认知盲点 cognitive_blindspot）：角色对自身行为有误解
- `ht_03_ext`（无意识互动 unconscious_interaction）：通过无意识小动作传递情感
- `metaphor_boundary`：比喻必须基于角色文化/认知背景，不允许跨阶级隐喻

**Write 约束更新：** `write_constraints.human_touch` 从 ht_01~ht_06 扩充为 ht_01~ht_12。  
**Self-Check 新增 Section F：** 六维审查框架，覆盖 show_dont_tell、密度检查、记忆碎片、渺小-宏大对比、人性缺口、节奏变奏。

---

### English Description

**File:** `config/human_touch_rules.yaml`  
**Modules affected:** `modules/writing/prompt.md`, `modules/interactive/prompt.md`

v8.2 had only ht_01 ~ ht_07. v8.4 adds five new rules, bringing the total to 12.

| Rule ID | Name | Core Requirement |
|---------|------|-----------------|
| ht_08 | Memory Fragmentation | Non-linear flashback triggered by sensory anchor, ≤200 chars |
| ht_09 | Mundane-Grand Contrast | At least once per 10,000 chars: map everyday detail to grand theme |
| ht_10 | Philosophical Metaphor | World-view insight emerges naturally from action/scene—no direct exposition |
| ht_11 | Indirect Humanity | Suggest unhealed wounds through avoidance behavior, never direct revelation |
| ht_12 | Motif Echo | An image/word planted early resonates at the climax for full-circle resonance |

Additional sub-rules added for dialogue subtext, cognitive blindspots, unconscious interactions, and metaphor boundary constraints.

**Write constraint updated:** `write_constraints.human_touch` expanded from ht_01~ht_06 → ht_01~ht_12.  
**Self-Check Section F added:** Six-dimension audit framework covering all new human-touch dimensions.

## <a id="2"></a>2. 导演模式 Director Mode（丁类）

### 中文说明

**配置文件：** `config/bangui_modes.yaml`、`config/trpg_immersion.yaml`  
**影响模块：** `modules/interactive/prompt.md`

帮回系统新增第四类（丁类）：**导演模式**。用户可以绕过选项机制，以导演身份直接发出场景级控制指令。

**触发方式：** `导演指令:` / `DM:` / `导演:` / `幕后:`

**支持指令：**

| 指令 | 语法 | 效果 |
|------|------|------|
| 场景注入 | `导演指令: 场景注入 {描述}` | 强制在下一回合开头呈现指定场景元素 |
| NPC 控制 | `导演指令: NPC {名字} {行为}` | 直接控制指定 NPC 的下一个行动或台词 |
| 节奏控制 | `导演指令: 节奏 {fast/slow/freeze}` | 强制切换决策密度 |
| 剧情跳过 | `导演指令: 跳过 {场景描述}` | 时间快进，跳过过渡场景 |
| 埋线 | `导演指令: 埋线 {伏笔内容}` | 在正文中悄悄植入伏笔，写入 KB 线索槽 |

**约束：**
- 导演模式不替用户做角色决策，只改变舞台和 NPC 状态
- 同一回合最多执行 2 条导演指令

**TRPG 新增功能（trpg_immersion.yaml）：**
- `dynamic_beat_mutation`：连续中性段落 > 3 次时触发节拍变奏（惊喜揭示/突发对抗等）
- `dynamic_temperature_drift`：场景压力驱动情绪温度漂移，每回合 ±2，每章 ±4
- `action_resolution`：统一的动作解析输出格式，含行动→结果→后果→钩子四步结构

**Foreshadowing Whisper 功能：** 
通过 `*[暗线指示: {内容}]*` 向 DM sub-agent 传递隐性伏笔指令，不展示给读者，自动写入 KB 线索槽。

---

### English Description

**Files:** `config/bangui_modes.yaml`, `config/trpg_immersion.yaml`  
**Module affected:** `modules/interactive/prompt.md`

A fourth 帮回 class (丁类) is added: **Director Mode**. Users can bypass the option-selection mechanism and issue scene-level control commands directly as a director.

**Trigger prefix:** `导演指令:` / `DM:` / `导演:` / `幕后:`

**Supported commands:** scene_inject, npc_control, tempo_control, plot_skip, foreshadow  
**Constraints:** Director Mode does not make character decisions for the player; it only alters the stage and NPC state. Maximum 2 director commands per turn.

**TRPG additions (trpg_immersion.yaml):**
- `dynamic_beat_mutation`: Auto-triggers a beat mutation when 3+ consecutive neutral beats are detected
- `dynamic_temperature_drift`: Scene pressure drives emotional temperature drift (±2/turn, ±4/chapter)
- `action_resolution`: Unified action-resolution output format with 4-step structure: action → result → consequence → hook

**Foreshadowing Whisper:** Accepts `*[暗线指示: {content}]*` syntax to pass hidden foreshadowing commands to the DM sub-agent; written to KB clue slot without being revealed to the player.

## <a id="3"></a>3. 跨模型一致性约束 T1/T2/T3 / Cross-Model Consistency

### 中文说明

**新增配置文件：** `config/cross_model_consistency.yaml`

为解决不同模型（Claude/GPT-4/Gemini）之间输出风格不一致的问题，新增独立配置文件，将约束分为三层：

| 层级 | 名称 | 内容 |
|------|------|------|
| T1 | 硬约束（Hard Constraints） | 禁止代理决策、禁止越权视角、禁止禁词、钩子强制、show-don't-tell 门槛 |
| T2 | 强引导（Soft Guidance） | 人味密度、感官落地、对话缺陷、tension-before-release、KB 同步、记忆碎片 |
| T3 | 创意层（Creative Layer） | 意象选择、句式风格等自由发挥区间 |

T1 违反 → 打回重写；T2 < 80% → 告警；T3 完全自由。

**输出格式锚点：** 章节输出、互动输出、质量报告三种格式均有明确锚点定义。  
**模型特化提示：** 针对 Claude（避免过度分析）、GPT-4（强化中文俗语）、Gemini（注意俗称列表）的特化指引。

---

### English Description

**New file:** `config/cross_model_consistency.yaml`

To address inconsistency across different AI models (Claude/GPT-4/Gemini), a dedicated consistency configuration is added with three constraint tiers:

- **T1 Hard Constraints:** 6 rules that must never be violated (proxy decision ban, POV lock, banned words, mandatory hook, show-don't-tell gate, etc.)
- **T2 Soft Guidance:** 6 target metrics with ≥80% compliance threshold (human-touch density, sensory grounding, dialogue imperfection, tension-before-release, KB sync, memory fragment presence)
- **T3 Creative Layer:** Free zone for model creativity (imagery, sentence style, etc.)

T1 violation → force rewrite; T2 < 80% → warning; T3 is unrestricted.

Includes output format anchors for chapter/interactive/quality_report modes, plus model-specific hints for Claude, GPT-4, and Gemini.

## <a id="4"></a>4. KB 自愈验证 / KB Auto-Healing

### 中文说明

**影响模块：** `modules/writing/prompt.md`

在写作模块的步骤 4.2 新增 **KB 自愈验证**。

在每章完成后：
1. orchestrator 加载当前 `state.json`，比对全部角色 / 物品 / 地点字段
2. 检测 KB Diff 中标注为 `status: pending` 的条目
3. 如发现已在正文落地的挂起条目，自动更新 `kb_template.json` 的 `status` → `confirmed`
4. 如发现正文与 KB 冲突（如角色位置矛盾），输出 `[KB-CONFLICT: ...]` 告警并建议修正方向

**效果：** 减少人工维护 KB 的工作量，提升长篇写作的世界观一致性。

---

### English Description

**Module affected:** `modules/writing/prompt.md`

Step 4.2 **KB Auto-Healing** is added to the writing module's post-chapter pipeline.

After each chapter completes:
1. Load `state.json` and compare all character/item/location fields
2. Detect `status: pending` entries in KB Diff
3. Auto-promote confirmed entries from `pending` → `confirmed` in `kb_template.json`
4. If a conflict is detected between chapter text and KB (e.g., character location mismatch), emit `[KB-CONFLICT: ...]` warning with a suggested fix direction

This reduces manual KB maintenance overhead and improves world-state consistency in long-form writing.

## <a id="5"></a>5. 人性化豁免预算 / Humanizer Exemption Budget

### 中文说明

**影响模块：** `modules/humanizer/prompt.md`

人性化模块（去 AI 痕迹）在 v8.4 新增豁免预算机制，防止过度统一化：

| 豁免类型 | 规则 |
|---------|------|
| R1 豁免 | 强烈语义对比场合（如角色自嘲、不完美对话），每 1000 字允许 1 次规则豁免，标注 `R1-EXEMPT` |
| R3 豁免 | 特定角色口癖 / 说话模式，每 2000 字允许 1 次，标注 `R3-EXEMPT` |
| 全局预算 | R1 + R3 豁免合计 ≤ 每章 3 次 |

**效果：** 在维持文风自然化的同时，保留角色语言个性，避免"过度净化"导致声音同质化。

---

### English Description

**Module affected:** `modules/humanizer/prompt.md`

The Humanizer module (de-AI-ification) gains an exemption budget system in v8.4 to prevent over-homogenization:

- **R1 Exemption:** Strong semantic contrast (self-deprecating dialog, intentionally imperfect speech) → 1 exemption per 1,000 chars, marked `R1-EXEMPT`
- **R3 Exemption:** Character-specific verbal tics / speech patterns → 1 exemption per 2,000 chars, marked `R3-EXEMPT`
- **Global Budget:** R1 + R3 combined ≤ 3 exemptions per chapter

This preserves character voice distinctiveness while maintaining the natural-language baseline, avoiding "over-sanitization" that flattens all characters to the same register.

## <a id="6"></a>6. LANDING 散文润色 / LANDING Prose Enhancement

### 中文说明

**影响模块：** `modules/interactive/prompt.md`

互动模式的 LANDING 流程（落盘）新增步骤 3.6：**散文润色**。

在将互动轮次合并为章节草稿后，自动执行以下流程：
1. 对 raw 对话段落按 ht_01~ht_12 检查人味密度
2. 对 < 60% 密度的段落执行散文化改写（while preserving 玩家的核心决策方向）
3. 输出改写标注：`[PROSE-ENHANCED: {改写原因}]`

**效果：** 解决互动模式下逐轮生成导致文本碎片化、人味不足的问题。

---

### English Description

**Module affected:** `modules/interactive/prompt.md`

The interactive mode's LANDING (chapter close-out) pipeline gains a new step 3.6: **Prose Enhancement**.

After merging interactive turns into a chapter draft:
1. Audit raw dialogue paragraphs against ht_01~ht_12 for human-touch density
2. Rewrite paragraphs scoring < 60% density into prose form (preserving the player's core decision direction)
3. Annotate rewrites with `[PROSE-ENHANCED: {reason}]`

This addresses paragraph fragmentation and low human-touch density that can occur when interactive turns are written one-by-one.

## <a id="7"></a>7. 冲突解析规则 / Conflict Resolutions

### 中文说明

v8.4 在多个配置文件中新增冲突解析规则，明确规定当不同规则之间存在优先级冲突时的处理方式。

**`config/emotional_temperature.yaml` 新增：**
- `conflict_resolutions.ht_03_vs_sensory_density`：ht_03（沉浸感）优先于 sensory_density（感官密度）限制；心理流动不计入感官密度上限
- `interactive_override`：互动模式由玩家行动驱动温度变化，不受 temperature_curve 硬约束；DM 可覆盖曲线

**`config/writing_rules.yaml` 新增（wr_11 内部）：**
- `ht_01_exception`：角色自嘲、不完美对话、叙述者微跑题场合，允许例外突破 ht_01 约束

**`config/style_rules.yaml` 新增：**
- `v8_4_conflict_resolutions.ht_07_vs_narrator_style`：ht_07（叙述者存在感）与 sd_04（短句节奏）冲突时，sd_04 优先；每段最多 1 个叙述者插入语
- `v8_4_conflict_resolutions.ht_03_vs_metaphor_ban`：ht_03（人性缺口）独白可使用"感知具象化"类比喻，但不得触犯 sd_09 明文禁止词

---

### English Description

v8.4 adds explicit conflict resolution rules to several config files, specifying priority order when rules clash.

**`config/emotional_temperature.yaml` additions:**
- `ht_03_vs_sensory_density`: ht_03 (immersion/interiority) takes priority over sensory_density limits; mental flow does not count toward the sensory density ceiling
- `interactive_override`: Player actions drive temperature changes in interactive mode, bypassing the static temperature_curve constraint; DM can override the curve

**`config/writing_rules.yaml` addition (inside wr_11):**
- `ht_01_exception`: Self-deprecating dialogue, intentionally imperfect speech, and slight narrator tangents are allowed to break ht_01 constraints

**`config/style_rules.yaml` addition:**
- `ht_07_vs_narrator_style`: When ht_07 (narrator presence) conflicts with sd_04 (punchy sentence rhythm), sd_04 takes priority; max 1 narrator interjection per paragraph
- `ht_03_vs_metaphor_ban`: ht_03 interiority monologue may use "perception-solidification" metaphors but must not trigger any sd_09 banned words
