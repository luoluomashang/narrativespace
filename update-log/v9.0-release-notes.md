# v9.0 Release Notes — 微观行文DNA采集系统 & 加载保障机制

**发布日期:** 2026-04-06
**版本:** 9.0.0
**上一版本:** 8.5.0

---

## 摘要 / Summary

v9.0 对**行文DNA采集系统**进行了根本性升级，从宏观统计维度深入到**语法构成层**（主谓宾定状补）和**微观语言学层**，同时修复了 `dna_human_*.yaml` 长期存在的加载缺口与优先级矛盾。

v9.0 rebuilds the **Writing DNA Collection System** from macro statistics down to **syntactic component level** (subject/predicate/object/modifier) and microscopic linguistic patterns, while fixing the long-standing `dna_human_*.yaml` loading gap and priority conflict.

## 核心变更 / Major Changes

### DNA采集系统 v1.0 → v2.0

旧版 DNA（v1.0）仅采集7个宏观统计段（句式节奏/词汇指纹/情绪曲线等），无法捕获语法构成模式、话语标记隐性化、零指代衔接等微观特征。

新版 DNA（v2.0）采集 **8个微观核心段 + 4个宏观补充段**：

| 基因段 | 层级 | 采集维度 |
|--------|------|---------|
| `sentence_architecture` | 微观 | **A. 语法骨架层**：句型分布（SVO/SV/倒装/无主句/连动/兼语）、主语类型、谓语复杂度、宾语处理、修饰策略；**B. 句式节奏层**：句长/方差/节奏模式/句首启动类型/断句偏好 |
| `lexical_structure` | 微观 | 四字格密度、动词名词化频率、叠词偏好、具象/抽象比、口语/书面比、标志词组、禁用词 |
| `paragraph_architecture` | 微观 | 段内逻辑模式、段首/尾句功能、段落节奏规律、密度调制模式 |
| `logic_structure` | 微观 | 因果链风格（显式/隐式）、论证推进方式、对比部署手法、信息分层策略 |
| `information_anchoring` | 微观 | 结果先行/过程先行比例、感官/认知前置顺序、已知→未知梯度、背景嵌入方式 |
| `discourse_marker_stealth` | 微观 | 显性标记密度、无标记转折占比、标记隐性化技法、段间衔接风格 |
| `cohesion_and_reference` | 微观 | 零指代频率、代词/名词/同义替换比例、指代链长度、主位推进模式 |
| `punctuation_and_pause` | 微观 | 逗号节奏功能、破折号用法、省略号功能、感叹号密度、断句偏好 |
| `emotion_curve_template` | 宏观补充 | 默认情绪曲线、高峰位置（保留兼容） |
| `dialogue_dna` | 宏观补充 | 轮次长度、连续轮数、密度（保留兼容） |
| `description_density` | 宏观补充 | 感官词密度、非视觉占比（保留兼容） |
| `transition_patterns` | 宏观补充 | 硬切/软切/蒙太奇比例（保留兼容） |

### `sentence_architecture` 深度：语法骨架层

专为中文商业网文设计的6维句型分类体系：

```yaml
dominant_sentence_pattern:
  svo_standard: 0.0      # SVO标准型（"他挥出一拳"）
  sv_no_object: 0.0      # SV无宾语型（"血雾散了"）
  inverted: 0.0          # 倒装型（"这种压力，他从未感受过"）
  subject_omitted: 0.0   # 无主句型（"猛地一震""抬手就是一刀"）
  serial_verb: 0.0       # 连动式（"他侧身闪过抬手格挡"）
  pivotal: 0.0           # 兼语式（"他让所有人闭嘴"）
```

此外包含主语类型偏好（代词/名词/零主语/拟人化）、谓语复杂度（单动词/连动/动补/动趋/状动组合）、宾语处理（直接/省略/双宾/宾语从句）、修饰策略（定语长度和位置、状语分布和前置率、补语密度、最大修饰层数）。

### 加载缺口修复

**问题**：旧版 `writing/prompt.md` 的动态风格加载只有4步，**没有 `dna_human_*.yaml` 扫描步骤**。即使 benchmark 模块生成了 DNA 文件，写作时也不会自动加载。

**修复**：在动态风格模块加载中新增 `dna_human_*.yaml` 显式扫描步骤，加载顺序调整为：

```
Step 1: active_modules（medium 优先级）
Step 2: dna_human_*.yaml 扫描（supreme 优先级）← 新增
Step 3: clone_*.yaml 扫描（high 优先级）
Step 4: global_author_dna（foundation 基底层）
Step 5: human_touch_rules + emotional_temperature
Step 6: cycle_quirks（tail，注意力权重最强）
```

### 优先级矛盾修复

**问题**：`index.yaml` 声明 `dna_human_*.yaml = supreme`，但 `writing/prompt.md` 将 `clone_*.yaml` 标注为"**优先级最高**"，形成矛盾。

**修复**：统一为6层标签体系，全系统一致：

```
cycle_quirks（tier-1）> dna_human_*（tier-2, supreme）> clone_*（tier-3, high）
> active_modules（tier-4, medium）> global_author_dna（tier-5, foundation）> 内置规则（tier-6, base）
```

## 新增 YAML 字段 / New YAML Fields

### `config/style_modules/index.yaml`

```yaml
dna_loading_guarantee:
  scan_per_chapter: true
  scan_path: ".xushikj/config/style_modules/"
  pattern: "dna_human_*.yaml"
  on_found: "立即加载并注入 write_constraints，优先级 supreme，在 clone_*.yaml 之前执行"
```

## 工程决策 / Engineering Decisions

### 为什么选择6维句型分类而不是更多？

中文商业网文高频出现的句型可归纳为：SVO标准型、SV无宾语型、倒装型、无主句型、连动式、兼语式。这6类覆盖了95%以上的实际句型分布，进一步细分的边际收益递减且难以统计。更重要的是，这6类对**风格克隆**有直接指导意义——连动式密度高说明动作节奏极快，无主句型偏高说明临场感强，这些正是商业网文读者感知差异的来源。

### 为什么宏观补充段"降级"而不是删除？

旧版 dna_human_*.yaml 文件已按旧 schema 生成，完全删除宏观段会破坏存量项目的兼容性。降级为"辅助参考"（confidence 独立计算，DO 指令权重低于微观段）是平滑过渡的最优解。新建项目可完全依赖微观8段，旧项目不受影响。

### 为什么 `dna_loading_guarantee` 放在 `index.yaml` 而不是 `meta_rules.yaml`？

`index.yaml` 是风格模块注册中心，写作 orchestrator 在加载任何风格文件前都会读取它。将 `dna_loading_guarantee` 放在 `index.yaml` 中，相当于在"守门人"处设置强制检查，比放在 `meta_rules.yaml`（执行时可能已完成加载）更早生效。
