# 场景执行模式（Pressure Unit 版）

本规则用于步骤8-9场景规划，目标是把“场景列表”升级为“可执行压力单元序列”。

## 1. 压力单元定义

一个压力单元（Pressure Unit）必须包含：

- POV 当下目标
- 阻碍/复杂化因素
- 单元末发生的变化
- 向下一单元传递的压力

没有“变化”就不是有效单元。

## 2. 每章建议 2-5 个单元

- 不按配额硬凑
- 高压章节可用更短更密的单元
- 关系章节可增加余波单元

## 3. Scene / Sequel 平衡

- Scene（正面冲突）适用于：战斗、追逐、博弈、调查突破
- Sequel（余波决策）适用于：关系后效、价值反思、策略重排

单章允许混合，但至少有一个单元承担“净变化”。

## 4. 模式切换：热度向 vs 口碑向

### 热度向（heat-forward）

- 入场更晚
- 解释更短
- 转折更快
- 章末拉力更强

### 口碑向（reputation-forward）

- 表层节奏可放慢
- 必须增加余波厚度
- 强化价值冲突与关系后效

### 双高模式

- 保留可读推进
- 同时留下主题后劲

## 5. 章节单元模板（可轮换）

常用组合：

1. 压力入场 -> 中段转折 -> 局部闭合 -> 残留债务
2. 正面对撞 -> 余波重排 -> 新承诺
3. 信息发现 -> 误判 -> 纠偏代价
4. 行动准备 -> 外部打断 -> 目标改写

## 6. 松结构章节的最低要求

即使是慢节奏章节，也必须至少满足一项：

- 关系状态改变
- 价值冲突变清晰
- 旧伤口/旧债被激活
- 既有主线承压上移

## 7. 场景规划复盘问题

- 第一单元是否进场过早
- 哪个单元是全章转折枢纽
- 关键冲突后是否给了余波
- 是否存在只搬运信息的空单元
- 相邻单元温度是否完全同质

## 8. 失败信号

- 所有单元温度一致
- 高压事件无后效
- 结尾戛然而止但无实质变化
- 单元很多但推进很少

## 9. 与 Layer-1 / Layer-2 的映射

- Layer-1 负责给出每章至少一个有效压力单元（可写门槛）
- Layer-2 在高风险章节补全 `tactical_beats` 与 `reaction_chain`
- 对标 RED/YELLOW 章节，优先补“中段转折”和“后效链条”字段
