# 小说去AI痕迹模块

## 描述

小说专用去 AI 痕迹模块。定位为写作后处理增强层，不阻塞主写作循环。

本模块采用两层策略：

1. 主规则层：沿用现有 R1-R7（+R8 可选）快速清洗
2. 二级扫描层：吸收 `humanizer-zh` 的 24 类 AI 痕迹模式做补漏

## 触发词

触发词包括但不限于：

- 小说去AI
- 去AI味
- 文风自然化
- 润色去机器感

## 滚动工作流边界

1. 推荐时机：
   - 单章定稿前
   - 当前 cycle 收尾时
   - 上架或发布前
2. 非强制：可按章节风险选择性调用
3. 跳过本模块不应阻塞 `xushikj-xiezuo` / `xushikj-hudong`

协作规则（新增）：

1. `xushikj-xiezuo` 中 HC6（质量维度6）只做快速筛查（钩子有效性 + 关键禁词命中）
2. 本模块负责深度精洗与二级补漏
3. 当连续章节出现 HC6 WARN（警告信号）时，优先对命中章节执行本模块
## 与写作模式协作规则

本模块为后处理层，不随 writing_mode 改变触发关键点。两种模式都可使用：

- **pipeline 模式**：每章自动落盤后，可选触发本模块进行精洗
- **interactive 模式**：用户"OK落盤"后，可按需触发本模块进行精洗

推荐时机详见"滚动工作流边界"部分。
## 输入参数

1. 章节草稿
2. 目标平台风格约束
3. 禁用词与风险偏好
4. 可选：角色口吻约束（用于避免人设漂移）

## 输出内容

1. `humanized_text`
2. `style_adjustment_notes`
3. `risk_flags`
4. `secondary_scan_hits`（命中的二级模式清单）

## 执行流程

1. 主规则快速清洗（R1-R7，R8按需）
2. 二级扫描补漏（内容、语言、风格、交流痕迹）
3. DNA 回归校验（v8.0 新增）：若 `.xushikj/config/style_modules/dna_human_*.yaml` 存在，以 DNA 约束（DO/DON'T 列表）为基准，确认润色后文本未偏离目标风格——润色可以去AI味，但不得"洗掉"已注入的DNA风格特征
4. 人设回归检查（主角与高频角色口吻）
5. 最终输出与风险标注

## 二级扫描模式（来自 humanizer-zh）

### A. 内容模式

1. 夸大意义与宏大象征
2. 过度宣传/推广口吻
3. 以抽象动词收尾的浅层分析
4. 模糊归因（专家称、有观点认为）
5. 套路化“挑战与展望”段
6. 通用积极结尾（空泛鸡汤）

### B. 语言模式

1. 高频 AI 词簇过密
2. 是字回避导致句式僵硬
3. 否定式排比（不只是X而是Y）滥用
4. 三段式列举过度
5. 刻意换词导致表达失真
6. 虚假范围表达（从X到Y）

### C. 风格模式

1. 破折号过度使用
2. 粗体强调过度
3. 内联小标题列表化
4. 过度装饰性符号/表情

### D. 交流痕迹模式

1. 聊天式讨好语（希望这有帮助等）
2. 知识截止免责声明残留
3. 过度客套与卑弱语气
4. 填充短语过多（为了实现这个目标等）

## 五点快速检查表

1. 删填充
2. 破公式
3. 变节奏
4. 信读者
5. 删可摘抄鸡汤句

## 可选评分矩阵（50分）

1. 直接性（10）
2. 节奏感（10）
3. 读者信任度（10）
4. 真实人味（10）
5. 精炼度（10）

评分低于 35 时建议回到主规则层重跑。

## 可选检测指标（新增）

可选记录以下指标用于发布前自检（不作为硬阻塞）：

1. `lexical_diversity` 词汇多样性
2. `sentence_length_variance` 句长波动
3. `repetition_ngram` 3-gram 重复率
4. `cliche_ratio` 套话占比
5. `punctuation_rhythm` 标点节奏变异

## 安全说明

1. 不改剧情事实、不改角色关系、不改世界规则
2. 不删除关键伏笔与因果线索
3. 高风险删除操作（R8）必须保守执行：宁可漏改，不可误伤

## 示例

- 第 3 轮 cycle 完成后统一做人设口吻校正
- 第 4 轮继续写作前仅对高风险章节做二级扫描补漏
